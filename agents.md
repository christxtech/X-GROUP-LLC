# AGENTS.md — Christ le Dieu du Tech

## Architecture Overview

Static HTML/JS frontend + Netlify Functions backend + Netlify Database (Postgres).

```
/
├── index.html          # Contact registration (home page)
├── admin.html          # Admin panel (password-protected)
├── social.html         # Social network + messaging
├── spin.html           # Spin game
├── links.html          # Social link exchange
├── view.html           # iframe viewer for external site
├── styles.css          # Shared global CSS
├── db/
│   ├── schema.ts       # Drizzle ORM schema (source of truth for DB)
│   └── index.ts        # DB client (drizzle-orm/netlify-db)
├── netlify/
│   ├── functions/      # API endpoints (TypeScript .mts)
│   │   ├── admin.mts   # Admin verify + all data reads + rights update
│   │   ├── contacts.mts
│   │   ├── users.mts   # Register, login, update, balance, debit, credit
│   │   ├── messages.mts
│   │   ├── spins.mts
│   │   └── links.mts
│   └── database/
│       └── migrations/ # Auto-generated by drizzle-kit generate
├── drizzle.config.ts   # Must output to netlify/database/migrations/
├── netlify.toml        # Build config (publish: ".")
└── tsconfig.json
```

## Key Design Decisions

- **No build step**: The publish directory is `"."` (root). All HTML files are served as-is.
- **Function routing**: Each function uses `config.path` for clean API URLs (e.g. `/api/users`).
- **Admin security**: Password is verified server-side by comparing SHA-256 hash — the hash is stored in `admin.mts`, never the plaintext password.
- **User auth**: Sessions stored in `sessionStorage` (client-side only). Password hashed with SHA-256.
- **Spin balance**: Stored in the `users.balance` column. Debit/credit via `/api/users?action=debit|credit`.
- **Message polling**: Chat auto-refreshes every 2 seconds via `setInterval`.
- **Link exchange rule**: Client-side check requiring at least one existing link to be clicked before publishing.

## Database Schema

Tables: `contacts`, `users`, `messages`, `spins`, `links`

After any schema change in `db/schema.ts`, run:
```bash
npx drizzle-kit generate
```
Never run migrations manually — Netlify applies them on deploy.

## Adding Features

- New API: add a `.mts` file in `netlify/functions/` with `export const config: Config = { path: '/api/...' }`
- New DB table: add to `db/schema.ts`, run `drizzle-kit generate`
- New page: add HTML file at root, add link to `<nav>` in all pages

## Coding Conventions

- Functions use standard Web API `Request`/`Response` (no Express)
- Import paths use `.js` extension even for `.ts` source files (ESM requirement)
- `Netlify.env.get()` for env vars, not `process.env`
- `drizzle-orm@beta` and `drizzle-kit@beta` required (Netlify DB adapter only on beta)
